# TestDashboard - 测试案例管理网页

## 功能需求

### 测试案例展示

#### 输入

| 字段 | 来源 | 说明 |
|------|------|------|
| 测试案例清单 | `data/tdd/第1期/自习SDD-登录功能-测试案例清单.md` | 包含 24 条用例的编号、测试点、前置条件、输入、预期结果 |
| 测试代码 | `test/第一期/自习SDD-登录功能测试.py` | pytest 实现，7 个测试类对应 7 个大模块 |

#### 输出

页面以**大模块 → 小模块**两级结构展示所有测试案例，每条用例包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| 大模块 | 字符串 | 功能测试的顶级分类，如"一、前端校验（输入为空）" |
| 编号 | 字符串 | 测试用例编号，如"TC-01" |
| 功能描述 | 字符串 | 中文说明，描述"测什么、怎么测、期望什么" |
| 测试状态 | 枚举 | 未运行 / 通过 / 失败 / 错误 / 跳过 |

**大模块清单：**

| 大模块 | 小模块 | 用例数 |
|--------|--------|--------|
| 一、前端校验（输入为空） | TC-01 ~ TC-03 | 3 |
| 二、手机号格式校验 | TC-04 ~ TC-08 | 5 |
| 三、正常登录 | TC-09 ~ TC-11 | 3 |
| 四、异常登录 | TC-12 ~ TC-14 | 3 |
| 五、账号锁定机制 | TC-15 ~ TC-21 | 7 |
| 六、系统异常 | TC-22 ~ TC-23 | 2 |
| 七、欢迎页 | TC-24 | 1 |

#### 核心逻辑

1. 页面加载时，读取测试案例清单，以大模块为分组、小模块为行，渲染为表格
2. 每个大模块为一个可折叠区块，区块标题显示大模块名称和该模块的通过率
3. 大模块旁显示状态徽章：全部通过（绿色）、全部失败（红色）、部分通过（黄色）、未运行（灰色）
4. 每条用例显示编号、功能描述、测试状态图标
5. 默认展开全部大模块；提供"展开全部"和"收起全部"操作
6. 首次加载且从未运行过测试时，所有用例状态显示为"未运行"

#### 边界条件

- 测试案例清单文档格式发生变化时，页面应能正确解析或提示解析失败
- 测试代码中的用例数量与清单文档不一致时，以清单文档为准展示
- 功能描述过长时应自动换行，不截断

### 一键测试

#### 输入

| 字段 | 来源 | 说明 |
|------|------|------|
| 用户点击 | 页面按钮 | "运行测试"按钮或各模块的"运行此模块"按钮 |
| 测试目录 | `test/第一期/` | pytest 运行的目标目录 |

#### 输出

**成功**：页面自动刷新，显示最新测试结果。

| 输出字段 | 说明 |
|----------|------|
| 各用例状态 | 更新为通过 / 失败 / 错误 / 跳过 |
| 汇总面板 | 更新总数、各状态计数、通过率 |
| 运行耗时 | 本次测试运行的总耗时 |
| 上次运行时间 | 更新为本次运行的时间 |

**失败**：测试运行过程中出现异常，页面提示错误信息。

| 场景 | 提示 |
|------|------|
| pytest 未安装 | "测试环境异常：pytest 未安装，请先执行 pip install pytest" |
| 测试代码有语法错误 | 展示 pytest 返回的错误信息 |
| 运行超时（超过 60 秒） | "测试运行超时，请检查测试代码" |

#### 核心逻辑

1. 用户点击"运行测试"按钮，按钮变为"运行中…"并显示加载动画，运行期间按钮置灰不可点击
2. 后端调用 pytest 运行 `test/第一期/` 目录下的所有测试
3. pytest 运行完成后，解析输出结果，提取每条用例的状态和失败详情
4. 前端获取最新结果，刷新页面展示
5. 用户点击"运行此模块"时，仅运行该大模块对应的测试类，其余逻辑与全部运行一致

#### 边界条件

- 测试运行期间用户再次点击按钮，应忽略（防重复提交）
- pytest 进程异常退出时，应捕获错误并友好提示

---

### 失败详情查看

#### 输入

| 字段 | 说明 |
|------|------|
| 用户点击 | 点击状态为"失败"或"错误"的用例行 |

#### 输出

展开显示该用例的错误堆栈信息，使用等宽字体、红色文字展示。

#### 核心逻辑

1. 仅失败和错误状态的用例可展开查看详情
2. 通过、跳过、未运行的用例不可展开
3. 点击已展开的详情行可收起

#### 边界条件

- 错误信息过长时，限制最大高度并提供滚动
- 错误信息中包含 HTML 特殊字符时应正确转义

## 汇总面板

#### 输出

页面顶部展示测试运行的概览信息：

| 字段 | 说明 |
|------|------|
| 总测试数 | 所有用例数量 |
| 通过数 | 状态为"通过"的用例数 |
| 失败数 | 状态为"失败"的用例数 |
| 错误/跳过数 | 状态为"错误"或"跳过"的用例数 |
| 通过率 | 通过数 / 总测试数，以百分比和进度条展示 |

## 非功能需求

| 项目 | 要求 |
|------|------|
| 启动方式 | 一条命令启动：`python xxx.py` |
| 访问方式 | 浏览器打开 `http://localhost:端口` |
| 依赖 | Python 3.8+、Flask、pytest |
| 前端 | 纯 HTML/CSS/JS，不引入前端框架 |
| 响应速度 | 页面加载 < 1秒 |
| UI 风格 | 暗色主题，深色背景 |
| 语言 | 所有界面文案使用中文 |

## 不在本次范围内

以下功能不属于本次需求范围，后续迭代另行规划：

- **多期次支持** — 自动发现"第二期""第三期"等新模块目录
- **历史趋势图** — 随时间变化的通过率曲线
- **CI/CD 集成** — 代码提交后自动运行并更新结果
- **在线编辑测试案例** — 在网页上修改测试描述或新增用例
- **用户认证** — 多用户登录、权限管理
- **测试用例覆盖率统计** — 代码行覆盖率、分支覆盖率
